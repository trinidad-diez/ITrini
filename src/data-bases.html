Structured Query Language

Data Modification Language

Used to query, edit, add and delete row-level data from database tables or views.

SELECT

SELECT [DISTINCT] * | expression [AS alias][,...] FROM table_name [WHERE conditions] [GROUP BY column_name, ...] [HAVING conditions] [ORDER BY column_name [ASC | DESC], ...];
  • Select all information from a table: SELECT * FROM table_name;
  • Select specific columns from a table: SELECT column1, column2 FROM table_name;
  • Select rows that meet a specific condition: SELECT * FROM table_name WHERE condition;
  • Select rows and group them by a specific column: SELECT column_name, COUNT(*) FROM table_name GROUP BY column_name;
  • Select rows and sort them by a specific column: SELECT * FROM table_name ORDER BY column_name ASC | DESC;
  • NULL behavior in operations: NULL is a data type (neither zero nor space)
  • Defining aliases for columns shown in the SELECT: AS
  • Concatenation of columns and literals: concatenate with || and literals with single quotes ' '
  • Including special characters using escape format: E''
  • Avoid duplicate records: DISTINCT
  • Limit/filter the number of records returned by a query, defining conditions with: WHERE
  • Comparison operators: =, >, <, >=, <=, BETWEEN, IN, LIKE, ILIKE, IS NULL
  • Defining search patterns with operators: %, _
  • Logical operators: AND, OR, NOT
  • Operator precedence rules
  • Sort records by columns: ORDER BY, ASC, DESC

    • Sorting by columns and by alias.

    • String manipulation functions: lower(), upper(), initcap()

  • GROUP BY clause

  • Joining tables with JOIN:
    • JOIN combines rows from two or more tables in a query, based on a specific condition.
      • If no condition is specified, a Cartesian product is returned.
      • Types of JOIN:
        • INNER JOIN: Returns rows that satisfy the ON condition.
        • LEFT OUTER JOIN: Returns all rows from the left table, and matching rows from the right table.
        • RIGHT OUTER JOIN: Returns all rows from the right table, and matching rows from the left table.
        • FULL OUTER JOIN: Returns all rows from both tables, with matches where available.
        • NATURAL JOIN: Joins tables based on columns with the same name and type, keeping only one copy of each common column.
        • CROSS JOIN: Returns the Cartesian product of both tables.
      • USING vs ON:
        • USING: Used when column names are identical; only one column is included in the result.
        • ON: Used for explicit join conditions; both columns are included in the result.
    • Subqueries:
      • A subquery is a SELECT statement nested inside another query, enclosed in parentheses.
      • Can be used in SELECT, INSERT, DELETE, UPDATE, CREATE TABLE AS, and more.
      • Types:
        • Scalar subquery: Returns a single value.
        • Multi-row subquery: Returns multiple values, used with IN, ANY, ALL, NOT.
        • Correlated subquery: References columns from the outer query and is evaluated for each row.
      • EXISTS / NOT EXISTS: Checks if a subquery returns any rows (returns TRUE/FALSE).
    • Set operators:
      • UNION: Combines results from multiple queries, removing duplicates.
      • UNION ALL: Combines results, including duplicates.
      • INTERSECT: Returns only rows present in both queries.
      • EXCEPT: Returns rows from the first query not present in the second.
    • Window functions:
      • Perform calculations across a set of rows related to the current row, without collapsing rows.
      • Categories:
        • Aggregation: MAX(), MIN(), AVG(), SUM(), COUNT()
        • Ranking: ROW_NUMBER(), RANK(), DENSE_RANK()
        • Analytic: LEAD(), LAG(), FIRST_VALUE(), LAST_VALUE(), NTH_VALUE(), NTILE()
        • Distribution: PERCENT_RANK(), CUME_DIST()
      • Syntax: window_function(args) OVER ([PARTITION BY ...] [ORDER BY ...])
    • Common Table Expressions (CTE):
      • Use WITH to define named subqueries for better readability and structure.
      • Syntax: WITH subquery_name AS (SELECT ...)
    • Conditional expressions:
      • CASE: Implements conditional logic in queries.
        • Syntax: CASE WHEN condition THEN result ... ELSE result END
      • COALESCE: Returns the first non-null argument.
        • Syntax: COALESCE(exp1, exp2, ...)
      • NULLIF: Returns NULL if two expressions are equal, otherwise returns the first expression.
        • Syntax: NULLIF(exp1, exp2)
Note:
  • SQL keywords are not case-sensitive, but it is a common convention to write them in uppercase for better readability.
  • Semicolons (;) are used to terminate SQL statements, especially when executing multiple statements in a single batch.
  • SQL comments can be added using double hyphens (--) for single-line comments or /* */ for multi-line comments.
  • Whitespace (spaces, tabs, and newlines) is generally ignored in SQL statements, allowing for flexible formatting.
  • Functions and operators are not the same: functions perform operations on values and return results, while operators manipulate values directly within expressions.
  • SQL is a declarative language: you specify what you want, not how to get it.
  • Most SQL implementations have vendor-specific extensions and differences.

  • Indexes can improve query performance but may slow down write operations.
  • Transactions allow you to group multiple operations into a single unit of work.
